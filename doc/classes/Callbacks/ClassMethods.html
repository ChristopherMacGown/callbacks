define_callbacks(*callbacks)

Defines one or more callback chains on a method. If the method does not exist or the callback type (before, outgoing, after, incoming) is invalid will raise CallbackError. This method implicitly defines a method of the same name as the callback chain which is then added to the list of callbacks on the method.

Examples of valid callbacks:

before_method_name:An example callback list called before the method.
outgoing_method_name:An example callback list before the method.
after_method_name:An example callback list called after the method.
incoming_method_name:An example callback list called after the method.

Execution Order:

When a chained method is called, the callbacks fire in the following order:

  1. before_ callbacks
  2. outgoing_ callbacks
  3. after_ callbacks
  4. incoming_ callbacks

Examples:

  # Implicitly defined before callback on some_method
  def before_some_method(*args)
    true
  end

  #Takes an optional list of methods and an optional block and adds each to the
  #list of callbacks on some_methods's before callback chain
  def self.before_some_method(*methods, &block)
    (methods << block).compact.each do |m|
      callbacks['some_method']['before'] << m
    end
  end

    def define_callbacks(*callbacks)
      callbacks.each do |callback|
        callback.to_s =~ /(#{CALLBACK_TYPES.join("|")})_(.*?)/
        type, method = [$1, $']

        raise CallbackError, "Invalid Callback Type" unless CALLBACK_TYPES.include? type
        raise CallbackError, "No such method" unless method_defined? method.to_sym

        self.callbacks[method] ||= Hash.new([])

        # Setting a default automatically means ||= short-circuits and all callbacks
        # end up added to the default array instead. Makes for an interesting bug.
        self.callbacks[method][type] = [] if self.callbacks[method][type].empty?
        self.callbacks[method][type] << callback.to_sym
